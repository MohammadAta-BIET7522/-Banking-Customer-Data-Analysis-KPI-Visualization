# 📊 Banking Customer Data Analysis & KPI Visualization

## 📌 Project Overview
This project performs **Banking Customer Data Analysis using Python**.  
The objective is to generate insights from banking data by creating **Key Performance Indicators (KPIs)** and visualizing them using different charts.

The project demonstrates the **complete data analysis workflow**, including:

- Data generation
- Data understanding
- Data cleaning
- KPI identification
- Data visualization
- Exporting results into a **PDF report**

This project is implemented using **Jupyter Notebook** and Python libraries.

---

# 🗂 Project Workflow

### Step 1: Create Banking Dataset
A mock banking dataset is generated containing:
- Customer ID
- Age
- Gender
- Account Balance
- Loan Amount
- Account Status
- Total Deposits
- Transaction Count

### Step 2: Install Required Packages
The following libraries are used:

- Pandas
- NumPy
- Matplotlib
- Seaborn
- Squarify
- YData Profiling
- ReportLab

---

### Step 3: Data Understanding
The dataset is analyzed using:
- Dataset shape
- Summary statistics
- Missing value detection
- Unique value analysis
- Distribution checks

---

### Step 4: Data Cleaning
Data cleaning techniques include:
- Handling missing values
- Correcting data types
- Removing duplicates

---

### Step 5: Data Visualization (KPIs)
Five different charts are created to represent banking KPIs:

| Chart Type | KPI |
|-------------|------|
| Bar Chart | Active vs Inactive Accounts |
| Line Chart | Average Transactions by Age |
| Scatter Plot | Account Balance vs Loan Amount |
| Pie Chart | Loan Distribution |
| Donut Chart | Account Status Distribution |

# 📈 Key Insights

- Most customers maintain **active accounts**.
- Customers with **higher balances often take larger loans**.
- Transaction frequency varies across age groups.
- A significant percentage of customers do not have loans.
